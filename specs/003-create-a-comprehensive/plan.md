## Summary
Enhancing the Trade Operations Dashboard to enable complete deal flow management with real seller location data, offer negotiation modals, and resumable operation state tracking. Focus on making the existing infrastructure fully functional with proper data persistence and UI interactions.

## Technical Context
**Language/Version**: TypeScript 5.x (Frontend), TypeScript/Node.js (Backend)  
**Primary Dependencies**: React Native + Expo, NestJS + Prisma, Google Maps API  
**Storage**: PostgreSQL with PostGIS extension for geospatial queries  
**Testing**: Jest for backend, React Native Testing Library for frontend  
**Target Platform**: React Native (iOS/Android), NestJS API server  
**Project Type**: web (frontend + backend architecture)  
**Performance Goals**: <500ms API response time, smooth map interactions at 60fps  
**Constraints**: Real-time offer updates via polling (5s intervals), Location accuracy ±100m  
**Scale/Scope**: ~100 concurrent admin users, ~1000 active trade operations

**User-Provided Requirements**:
1. Ensure real seller entities with proper location data for map markers
2. Implement offer modal when "Send Offer" button is clicked
3. Make active operations resumable with full state visibility
4. Show sent offers, acceptances, and counter-offers in operation details
5. Fix non-functional trade operations dashboard

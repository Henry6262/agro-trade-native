# Rules Overview

**Purpose**: Centralize all enforced coding standards so every agent reads the relevant rule set *before* planning or writing code.

## How to Use
1. Start here at the root `rules/README.md`.
2. Drill into the stack-specific folder you are about to touch (currently `rules/frontend/` for the mobile app).
3. Read the `README.md` inside that folder and every linked rule file (structure, components, logic, etc.).
4. Only then begin Plan Mode or implementation.

## Available Rule Sets
- `rules/frontend/` – React Native mobile application (Expo) rules.
- _More stacks coming soon (backend, admin dashboard)._

Always keep these documents in sync with architectural decisions (handbook, blueprints, status tracker).

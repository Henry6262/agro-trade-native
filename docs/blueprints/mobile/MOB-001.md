# MOB-001 Blueprint – Mobile Onboarding Refactor

**Epic**: Mobile Experience  
**Story**: Refactor onboarding into Page → Section → Feature hierarchy  
**Status**: Not Started  
**Last Updated**: 2025-11-15

---

## Goal
Replace the current monolithic onboarding screens with structured, reusable modules (Page → Section → Feature) that follow the mobile architecture rules. Improve state management, API integration, and UI consistency.

---

## Acceptance Criteria
1. Each onboarding role (Seller/Buyer/Transport) has its own page folder with sections (e.g., Flow steps, Summary, Account).
2. Every section and feature uses the required folder structure (`components/`, `hooks/`, `service.ts`, etc.).
3. Hooks fetch data via services; components use only design-system primitives.
4. Onboarding state stored in feature-level Zustand stores; global progress managed centrally.
5. Google Sign-In integration uses the new service + hooks.
6. Tests exist for hooks/services (React Native Testing Library + MSW).
7. Documentation updated (`docs/handbook/projects/mobile.md` + new page doc).

---

## Components & Files
| Feature | Files |
|---------|-------|
| SellerFlow | `pages/Onboarding/pages/Seller/sections/Flow/features/ProductSelection/…` |
| BuyerFlow | Same structure as Seller |
| TransportFlow | Same structure as Seller |
| AuthSection | `features/GoogleSignIn/{components/hooks/service}` |
| SummarySection | `features/SummaryCard` |

---

## Data Flow
1. User selects role → global onboarding store sets `role`.
2. Page renders role-specific sections (Flow, Requirements, Summary).
3. Each section uses feature hooks (e.g., `useProductSelection`) to load data from services.
4. Services call backend endpoints (`/mobile/onboarding/*`).
5. Hooks format/validate data and expose callbacks.
6. Components render using `app/components/ui` primitives.

---

## Testing Plan
- Hook tests using React Native Testing Library + MSW.
- Service tests mocking API client.
- Integration test per role to ensure flow completes and triggers navigation.

---

## Rollout
- Feature flag `onboarding_refactor`.
- Migrate existing onboarding state to new store.
- Monitor analytics for step completion.
- Roll back by toggling flag if needed.
